# Contributing

Thanks for helping improve this book. This guide keeps contributions fast and consistent.

## Quick Start

1. Create a branch for your change.
2. Edit the Markdown in `docs/`.
3. Run a local build if possible.
4. Open a pull request with a clear summary.

## What to Contribute

- Fix typos or broken links
- Add references or clarifications
- Improve examples or test cases
- Report issues you can’t fix

## Style Notes

- Keep the tone practical and plain English.
- Use short paragraphs and concrete examples.
- Add links for papers and standards when available.
- Avoid adding raw URLs inside sentences unless they are references.

## Local Build

```bash
pip install -r requirements.txt
mkdocs serve
```
